The raw dataset consists of 3290 restaurants with under 30 minutes delivery time using the iconic place address in downtown for each populous city in Canada (Food Delivery Across Canada (Door Dash), 2022). There are 10 variables including categorical(e.g., restuarant, url, city, price_range, category_1, category_2), numerical(e.g., #, distance, star, num_reviews). This data is collected from Door Dash website(https://www.doordash.com/)

  • #: (double) index of the data
  • restaurant(character type) the name of the restaurant on Door Dash.
  • url: (character type) restaurant menu page on url on Door Dash.
  • distance: (double) distance from the place to make an order (mile).
  • star: (double) How favorable out of 5 stars.
  • num_reviews: (double) number of reviews.
  • city: (Character type) city where an order placed.
  • price_range: (Character type) average price range of the restaurant.
  • category_1: (Character type) category of the cuisine of the restaurant.
  • category_2: (Character type) category of the food ordered in the correspond restaurant.

Research Question¶

The research aims to which factors influence restaurant rating(star) on Door Dash. Specifically, we seek to determine how (explanatory) variables like distance, number of reviews and city affect a restaurant's average customer rating (response).

This study focuses on both inference and prediction:

  • Inference: Understanding the key factors that drive restaurant ratings.

  • Prediction: Developing a model that can estimate a new restaurant’s rating based on its attributes.

Since we aim to evaluate the determinants of restaurant ratings, certain columns such as index, restaurant name, and URL do not provide meaningful information for the model. Therefore, they will be omitted from this study. Additionally, we found that the distribution of restaurants across different price_range levels is highly uneven, which may introduce bias into the analysis. To avoid this issue, we will exclude price_range from the model. Furthermore, category_1 and category_2 contain too many unique levels, which would complicate the model and make interpretation difficult. As a result, these variables will also be excluded. Moreover, we classified cities into two groups evenly based on their population: small cities ($x_i = 0$) and big cities ($x_i = 1$). Cities like Vancouver, Calgary, Toronto, and Montreal will be considered big cities and the rest will be small cities.

Methods and Plans¶

Summary We aim to identify the best combination of inputs that yield the best prediction using stepwise selection with linear regression. Then we will analysis the model performance with residual plot and Q-Q plot, accessing accuracy and prediction.

Why this method?¶

Interaction Terms

  • First, we will add some interaction terms into the model to guarantee we will not miss any potential information between features.

Stepwise Selection

  • Then, we use backward selection to filter true features that really matter. This method produces the simple regression model, which enhances interpretability.
  • The process is deterministic and provides clear and consistent results.

Linear Regression

  • Linear regression is necessary because it is highly interpretable. The coefficient $\beta_j$ directly represent the change in $Y_i$ for a unit change in $x_i$.
  • Interaction terms can be explicitly added to the model.
  • Linear regression is computationally efficient and straightforward to implement.

Assumptionsare essential for the method selected:

Applying Linear Least Square Regression requires several assumptions to ensure validity:

  • Linearity: The expected value of the response variable (star rating) is a linear combination of the explanatory variables.
  • Independence: The errors are independent, i.e., $\mathbb{E}[\epsilon] = 0$.
  • Normality: The errors (not the response itself) are normally distributed at each level of the covariates.
  • Equal Variance: The standard deviation of the responses is equal for all levels of covariates (homoscedasticity).
  • Adequate Sample Size: Assumes a sufficiently large sample size to ensure reliable estimation of the model parameters.

Potential limitation and weaknesses

  • Interaction terms may introduce multicollinearity, leading to unstable estimates.
  • Stepwise selection is a greedy method that can overfit the training data; cross-validation is necessary to assess model performance.
  • Including many interaction terms can reduce model interpretability.
  • The model may be limited by the data itself, as star ratings can be high even with few reviews, suggesting nonlinearity or unobserved factors not captured by the available features.

Implementation of a proposed model: computational code and output¶

We will begin by performing variable selection using backward stepwise selection to identify the model with the smallest Cp, representing the most parsimonious model that balances goodness-of-fit and predictive power, from the 6 input variables for prediction:

set.seed(42)
# Split the data
data_split <- initial_split(cleaned_data, prop = 0.7, strata = star)
training_set <- training(data_split)
testing_set <- testing(data_split)

# Applied subset selection
feature_selection_bw <- regsubsets(star ~ (num_reviews + distance + city_size)^2,
                               data = training_set,
                               nvmax = 6,
                               method = "backward")
FS_summary <- summary(feature_selection_bw)

# The model size with the smallest Cp
cp_min <- which.min(FS_summary$cp)

# Get the variables present in the model with smallest Cp
selected_var <- names(which(FS_summary$which[cp_min, -1]))

cat("Table: Variables included in the model with the smallest Cp\n")
print(selected_var)

# Use the selected feature to train the model
MLR_model <- lm(star ~ (log(num_reviews) + distance + city_size + num_reviews*city_size + distance*city_size), data = training_set)
tidy(MLR_model)

# Test the model
test_prediction <- testing_set %>%
    add_predictions(MLR_model, var = "pred_OLS")

# Compute the metrics table
test_metric <- test_prediction %>%
    metrics(truth = star, estimate = pred_OLS)
cat("Table: Model coefficients and performance metrics\n")
test_metric
